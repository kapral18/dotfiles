# Docusaurus website

Public docs site for [`kapral18/dotfiles`](https://github.com/kapral18/dotfiles), serving the markdown under [`../docs/`](../docs/) at <https://kapral18.github.io/dotfiles/>.

## Local development

From the repo root:

```bash
make docs           # start the dev server at http://localhost:3000/dotfiles/
make docs-build     # production build into website/build/
make docs-serve     # serve the production build locally (matches GitHub Pages)
make docs-clean     # clear the .docusaurus/ cache
```

The root `package.json` exposes the same flow for npm users:

```bash
npm run docs
npm run docs:build
npm run docs:serve
npm run docs:clean
```

The `make` targets and npm scripts auto-install `website/node_modules/` on first run via the global Yarn Classic 1.22.22 binary and `--frozen-lockfile`. The dev server hot-reloads on changes under `docs/` and `website/`.

If you'd rather drive Yarn directly, the equivalent is `yarn --cwd website {start,build,serve,clear}` from the repo root, or `cd website && yarn <script>`.

### Yarn version

The website uses the global Yarn Classic binary (`yarn --version` should report `1.22.x`) and records that expectation in the `packageManager` field in [`package.json`](package.json). If `yarn.lock` changes unexpectedly, rerun the install with Yarn 1 instead of a Yarn 3/4 binary.

## Deployment

Pushing to `main` runs `.github/workflows/deploy-docs.yml`, which builds and uploads the static site to GitHub Pages. The workflow only runs when files under `docs/`, `website/`, or the workflow itself change.

## Architecture

- Markdown source-of-truth lives in [`../docs/`](../docs/) — do not move it. Docusaurus reads it via `path: '../docs'` in `docusaurus.config.ts`.
- `markdown.format: 'md'` is set so files using `<placeholder>` syntax don't trip the MDX parser.
- Sidebar is auto-generated from the `docs/` directory tree. Use `_category_.json` files in folders to override labels or order.
- Cross-repo links (e.g. `../home/...`) are rewritten to absolute GitHub URLs at build time by [`src/remark/cross-repo-links.js`](src/remark/cross-repo-links.js).
- This directory is outside `home/`, so chezmoi does not manage it.

### Search

DIY local search, no external service:

- [`src/plugins/local-search`](src/plugins/local-search/index.ts) — Docusaurus plugin. On `loadContent`, walks `../docs/`, splits each file at `##` boundaries, and builds a [MiniSearch](https://lucaong.github.io/minisearch/) index with **one document per section**. Anchors are generated by [`github-slugger`](https://github.com/Flet/github-slugger) so the URL stored in the index (e.g. `#1password`) matches the heading ID Docusaurus emits. Writes `static/search-index.json` (~227 KB for ~314 sections across ~40 pages).
- [`src/theme/SearchBar`](src/theme/SearchBar/index.tsx) — swizzled SearchBar slot. Renders the `⌘ K` / `Ctrl K` trigger in the navbar and a modal with keyboard navigation. Each result shows the section title with the page title as a breadcrumb. Lazy-fetches the index on first open. `Esc` closes, `↑↓` navigate, `Enter` opens, `/` also opens search when not typing in another input. Field weights: `title 4×`, `pageTitle 3×`, `headings 2×`, `body 1×`.

`static/search-index.json` is gitignored — it's rebuilt every `yarn build` / `yarn start`. Edits to docs during dev mode do **not** auto-refresh the index; restart the dev server (`make docs`) to reindex.
