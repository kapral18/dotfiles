# Elastic Drafting Guidance

Use this for Elastic Stack repos (Kibana / Elasticsearch / related).

Templates:

- PR (default): `~/.agents/github_drafting/elastic/templates/pr.template.md`
- PR (bugfix):
  `~/.agents/github_drafting/elastic/templates/pr_bugfix.template.md`
- PR (chore/migration):
  `~/.agents/github_drafting/elastic/templates/pr_chore.template.md`
- PR (feature):
  `~/.agents/github_drafting/elastic/templates/pr_feature.template.md`
- Issue: `~/.agents/github_drafting/elastic/templates/issue.template.md`

Ownership:

- For Kibana Management-owned areas, consult
  `~/.agents/github_drafting/elastic/team_kibana_management.md` and the repo
  CODEOWNERS to propose `Team:Kibana Management`.

Labels:

- Use `~/.agents/github_drafting/elastic/labels.md` as propose-only guidance.
- Always propose labels first and get explicit confirmation before applying.
