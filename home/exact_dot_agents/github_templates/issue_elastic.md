# Issue Body (Elastic / Kibana / Elasticsearch)

Use this when reporting bugs/requests in Elastic Stack codebases.

Template:
```
## Problem

## Expected

## Actual

## Reproduction

## Environment
- Stack version:
- Deployment (cloud/on-prem):
- Browser/OS (if UI):

## Notes
- Logs / screenshots / sample docs (redact secrets)
- Related issues/PRs

## Labels / Meta (agent guidance)

Common label patterns observed in `elastic/kibana`:

- Type/intent: `bug`, `chore`, `help wanted`, `review`, `blocked`, `feedback_needed`, `tests_needed`
- Team ownership: `Team:<name>`
- Feature tags: `Feature:<area>` or `feature:<area>`
- Priority/severity sometimes appears as: `blocker`

Use these to propose (not apply) triage suggestions.
```
